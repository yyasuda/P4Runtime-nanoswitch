## Tutorial 0: Preparing the Environment

Each tutorial includes files such as p4info.txt generated by compilation. If you compile the P4 switch program yourself, please provide P4Runtime-compatible options as shown below.

```bash
root@f53fc79201b8:/tmp# p4c --target bmv2 --arch v1model --p4runtime-files p4info.txt nanosw01.p4 
root@f53fc79201b8:/tmp# ls
nanosw01.json  nanosw01.p4  nanosw01.p4i  p4info.txt
root@f53fc79201b8:/tmp# 
```

The generated p4info.txt and nanosw01.json will be used later when starting the P4Runtime Shell.

#### Starting the Mininet Environment

Here, we use the [P4Runtime-enabled Mininet Docker Image](https://hub.docker.com/repository/docker/yutakayasuda/p4mn) as the switch. You can start it as follows.

Start a Mininet environment with P4Runtime support in a Docker environment. Note that the --arp and --mac options are specified at startup so that ping tests can be performed without ARP processing.

```bash
$ docker run --privileged --rm -it -p 50001:50001 -e IPV6=false yutakayasuda/p4mn --arp --topo single,3 --mac
(snip...)
*** Starting CLI:
mininet> 
```

You can confirm that port 1 of s1 is connected to h1, port 2 to h2, and port 3 to h3.

```bash
mininet> net
h1 h1-eth0:s1-eth1
h2 h2-eth0:s1-eth2
h3 h3-eth0:s1-eth3
s1 lo:  s1-eth1:h1-eth0 s1-eth2:h2-eth0 s1-eth3:h3-eth0
mininet> 
```

The MAC address of the interface h1-eth0, which connects h1 to the switch, is 00:00:00:00:00:01. Similarly, h2 is 00:00:00:00:00:02, and h3 is 00:00:00:00:00:03.

### Connecting P4Runtime Shell and Mininet

From this point on, experiments will be conducted by synchronizing the host directory /tmp/P4runtime-nanoswitch with /tmp inside the Docker container. Create the directory for this purpose and place the files for each tutorial step there.

```bash
$ mkdir /tmp/P4runtime-nanoswitch
$ cp -rp nanosw0* /tmp/P4runtime-nanoswitch
$ ls /tmp/P4runtime-nanoswitch
nanosw01	nanosw02	nanosw03	nanosw04	nanosw05	nanosw06
$ 
```

In this state, start the P4Runtime Shell as shown below. This example is for trying the nanosw01 tutorial. Adjust the IP address to match your environment.

```bash
$ docker run -ti -v /tmp/P4runtime-nanoswitch:/tmp p4lang/p4runtime-sh --grpc-addr 192.168.1.2:50001 --device-id 1 --election-id 0,1 --config /tmp/nanosw01/p4info.txt,/tmp/nanosw01/nanosw01.json
*** Welcome to the IPython shell for P4Runtime ***
P4Runtime sh >>>
```

Verify that simple commands such as tables work as shown below to confirm that the connection with Mininet is established correctly.

```bash
P4Runtime sh >>> tables
MyIngress.l2_match_table

P4Runtime sh >>>
```

This completes the preparation.

** Note for ARM Mac: ** To run P4Runtime Shell on an ARM-based Mac, it is currently necessary to enable Rosetta. In Docker Hub settings >> General >> Virtual Machine Options, check “Use Rosetta for x86_64/amd64 emulation on Apple Silicon”. Then it is recommended to add the platform option to the docker command, such as ```$ docker run --platform=linux/amd64 ...```.
