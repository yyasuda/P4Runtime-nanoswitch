## Tutorial 6: NanoSwitch06

It takes time to create a flow entry in response to a Packet-In generated by a packet destined for a broadcast address or an unknown address. If another packet to the same address arrives before the entry is created, the controller will attempt to INSERT twice and cause an error. A typical case would be two broadcast packets sent consecutively from a host. This is a common situation, such as when ARP Requests for two hosts are issued.

There are various possible workarounds, but here we apply the simplest one. In other words, when adding a flow entry, if duplicate registration occurs, the error is ignored.

### Experiment

#### double_send.py

Below is the Python code, double_send.py, that sends two broadcast packets consecutively. By adjusting the sleep() time in the code, you can also examine how long it takes to complete the flow entry setup. As you can see from the destination interface and srcMac, this program generates packets equivalent to those sent from the h1 host.

```python
import socket
import time
iface = 'h1-eth0'
dstMac = b'\xff\xff\xff\xff\xff\xff'
srcMac = b'\x00\x00\x00\x00\x00\x01'
proto = b'\x88\xb5' # protocol type is IEEE Local Experimental
packet = dstMac + srcMac + proto + b"012345678901234567890123456789012345678901234567890123456789"
s = socket.socket(socket.AF_PACKET, socket.SOCK_RAW, socket.htons(3)) # 3:ETH_P_ALL
s.bind((iface, 0))
print('send #1 packet.')
s.send(packet)
time.sleep(1)
print('send #2 packet.')
s.send(packet)
print('done.')
s.close()
```

In this case, the second packet arrives at the switch before the flow entry created by the first packet’s Packet-In processing is fully registered. As a result, the second packet is also sent as Packet-In, and the process of registering the same flow entry twice is executed. However, the write() operation fails, causing a P4RuntimeWriteException.

### Error handling

There are various possible solutions, but here we apply the simplest one. That is, when adding a flow entry, if duplicate registration occurs, the error is ignored.

#### tutorial.py

The error occurs in the insertFlowEntry() function in tutorial.py. Modify the following part. Originally, it was just a single line:

```python
def insertFlowEntry(dstMac, srcMac, port):
    ...(snip)...
    client.write(req)
```

Add error handling as follows:

```python
    try:
        client.write(req)
    except P4RuntimeWriteException as e:
        if e.errors[0][1].canonical_code == code_pb2.ALREADY_EXISTS:
            print("Flow already exists. ignoring.")
        else:
            print(e)
            raise e
```

In other words, for errors caused by duplicate registration, simply print a message and ignore them. For errors other than ALREADY_EXISTS, raise them as usual. To add this code, include the following two lines at the beginning of tutorial.py:

```python
from p4runtime_sh.p4runtime import P4RuntimeWriteException
from google.rpc import status_pb2, code_pb2 
```
